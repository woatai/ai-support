# 跨境电商 AI 客服助手项目

## 项目定位

项目名称：跨境电商 AI 客服助手

项目方向：AI 应用开发 / 企业提效工具 / 智能客服

项目目标：

基于跨境电商客服场景，构建一个 AI 客服辅助工具，帮助客服人员快速处理商品咨询、物流查询、退款售后、优惠券问题和投诉安抚等高频重复问题。系统通过大模型 API 生成客服回复，结合知识库检索和模拟订单数据，提供引用来源、订单信息和转人工建议。

这个项目不是单纯聊天机器人，而是一个面向真实业务流程的 AI 客服工作台。

## 为什么选择这个项目

选择这个项目的原因：

1. 结合个人经历：之前做过客服，也有软件测试、接口测试、问题定位和缺陷闭环经验，能理解客服和业务流程中的重复工作。
2. 贴合 AI 应用岗位：项目涉及 LLM API、Prompt 工程、RAG 知识库、业务流程自动化、数据记录和前后端接口。
3. 企业受众较广：客服、运营、售后、电商团队都存在大量重复问答和规则判断场景。
4. 比普通知识库问答更有业务价值：除了回答问题，还能结合订单、物流、售后规则和转人工判断。

## 初版核心流程

```text
用户输入客服问题
↓
系统识别问题类型
↓
检索 FAQ / 商品 / 物流 / 售后知识库
↓
如果问题涉及订单，查询模拟订单数据
↓
调用大模型生成客服回复
↓
展示引用来源、订单卡片、处理建议和转人工建议
↓
保存会话记录
```

## 初版功能模块

### 1. AI 客服对话

用户输入客户问题，系统生成客服回复。

示例问题：

```text
我的订单什么时候发货？
为什么物流一直没有更新？
这个商品支持退货吗？
优惠券为什么用不了？
我收到的商品有质量问题怎么办？
```

初版要求：

- 回复礼貌、简洁、专业；
- 不编造订单、物流、退款状态；
- 信息不足时，引导用户提供订单号、商品名或问题截图；
- 对投诉类问题给出安抚语气。

### 2. 意图分类

将用户问题分类为不同客服场景。

分类建议：

```text
商品咨询
物流查询
退款售后
订单问题
优惠券问题
投诉安抚
其他问题
```

用途：

- 决定使用哪套 Prompt 模板；
- 决定是否查询订单数据；
- 决定是否需要转人工。

### 3. RAG 知识库问答

上传客服相关资料，构建知识库。

知识库资料建议：

```text
商品 FAQ
物流时效规则
退款退货规则
优惠券使用规则
订单状态说明
客服话术规范
投诉处理 SOP
```

实现目标：

- 支持上传 `.txt` / `.md` 文档；
- 对文档进行文本切分；
- 存入向量数据库；
- 用户提问时检索相关片段；
- 回答时展示引用来源；
- 无相关资料时不编造答案。

### 4. 模拟订单查询

使用 SQLite 存储模拟订单数据。

订单字段建议：

```text
订单号
用户昵称
商品名称
订单状态
支付状态
发货状态
物流单号
物流状态
退款状态
下单时间
更新时间
```

用途：

- 用户提供订单号后，系统返回订单卡片；
- AI 回复时结合订单状态生成更准确的话术；
- 物流异常、退款中、未发货等情况可触发转人工建议。

### 5. 转人工建议

系统根据规则判断是否建议人工客服介入。

触发条件示例：

```text
知识库没有相关答案
用户情绪明显不满
订单状态异常
物流长时间未更新
退款超过承诺时间
用户要求投诉或赔偿
```

输出内容：

```text
是否建议转人工：是 / 否
原因：物流超过预计时效，需要人工进一步核实
建议处理：安抚用户并创建售后工单
```

### 6. 会话记录

保存每次客服对话，便于复盘和展示项目完整性。

记录字段建议：

```text
用户问题
AI 回复
问题分类
引用来源
订单号
是否建议转人工
模型名称
耗时
创建时间
```

## 第一阶段最小闭环

先完成最小可运行版本：

```text
用户在 Streamlit 页面输入问题
↓
Streamlit 调用 FastAPI `/chat` 接口
↓
FastAPI 调用大模型 API
↓
大模型根据客服 Prompt 生成回复
↓
FastAPI 返回 JSON
↓
Streamlit 展示 AI 回复
```

第一阶段只做普通客服回复，不做 RAG、不做订单查询。

验收标准：

```text
页面能打开
能输入客服问题
点击发送后能返回 AI 回复
后端 `/chat` 接口能正常返回 JSON
README 记录启动方式
```

## 一个月开发路线

### 第 1 周：普通客服回复

目标：跑通 AI 客服对话最小闭环。

## 简历项目描述

项目名称：

```text
跨境电商 AI 客服助手｜AI 应用开发
```

项目描述：

```text
该项目面向跨境电商客服场景，支持基于商品 FAQ、物流规则、退款售后规则和订单数据生成客服回复。系统通过 RAG 知识库检索相关业务资料，结合大模型生成回复内容，并展示引用来源、订单卡片和转人工建议，帮助客服提升回复效率和回复准确性。
```

项目职责：

```text
1. 基于 Python + FastAPI 搭建后端服务，封装客服对话、知识库问答、订单查询和会话记录接口；
2. 接入 DeepSeek / OpenAI 大模型 API，设计客服回复 Prompt 模板，约束回复语气、输出格式、引用来源和兜底规则；
3. 基于 Chroma 实现 RAG 知识库，支持 FAQ、物流规则、退款规则、商品资料上传、文本切分、向量入库和相似度检索；
4. 使用 SQLite 构建模拟订单数据，支持根据订单号查询订单状态、物流状态和退款状态，并生成订单信息卡片；
5. 实现用户问题意图分类，区分商品咨询、物流查询、退款售后、优惠券、投诉等场景；
6. 设计转人工判断规则，当知识库无结果、订单异常或用户投诉时提示人工介入；
7. 使用 Streamlit 搭建客服工作台，展示 AI 回复、引用来源、订单信息和处理建议。
```

项目亮点：

```text
1. 项目结合客服与测试经历，围绕电商售前售后高频重复问题设计 AI 提效流程；
2. 不只是普通聊天机器人，而是结合知识库、订单数据、意图分类和人工转接建议的客服工作台；
3. 使用 RAG 降低模型胡编风险，回答内容可追溯到具体业务资料；
4. 通过 Prompt 模板约束客服回复语气、格式和业务边界，提高回复稳定性；
5. 具备后续扩展多渠道客服、工单系统、商品卡片和物流接口的空间。
```
